import { Steps } from '@astrojs/starlight/components';

<Steps>

1. **Create a new project**

   ```bash
   devify init my-app
   ```

   The CLI walks you through an interactive setup:

   - **Database** — PostgreSQL (default), MySQL, SQLite, MongoDB, or none
   - **Cache** — In-memory (default), Redis, or BadgerDB
   - **Mail** — SMTP (default, works with Mailpit), Resend, SendGrid, Postmark, SES, or none
   - **Storage** — Local (default), S3, MinIO, SFTP, or WebDAV
   - **Renderer** — Templ (default) or Go templates
   - **Modules** — Multi-select checklist (auth pre-selected). Dependencies are resolved automatically — selecting billing auto-adds auth + tenant.
   - **OAuth** — If auth is selected: Google, GitHub, Discord (optional)
   - **PWA** — If PWA module is selected: offline, install prompt, camera, GPS, push notifications, and more
   - **Frontend** — None, HTMX + Templ, React, Vue, Svelte, or SolidJS

   After confirming your selections, the CLI scaffolds a fully wired project — `main.go`, config, `compose.yml`, `Makefile`, and starter UI.

2. **Navigate into your project**

   ```bash
   cd my-app
   ```

3. **Start services and the dev server**

   The project includes a generated `compose.yml` and `Makefile`. One command starts everything:

   ```bash
   make dev
   ```

   This starts your database container (PostgreSQL by default), Mailpit for local email testing, and the Air hot-reload server. Your app is running at `http://localhost:4000`.

   If you chose a SPA frontend (React, Vue, Svelte, SolidJS), use `make dev-all` to start both the Go backend and the frontend dev server.

   :::tip
   The `Makefile` auto-detects Podman or Docker — no configuration needed.
   :::

4. **Run migrations**

   In a separate terminal:

   ```bash
   devify migrate
   ```

   This runs all pending migrations in dependency order — auth tables first, then RBAC, tenant, and so on.

6. **Create your first module**

   ```bash
   devify module:create blog --routes --migrations --config
   ```

   This scaffolds a complete module at `modules/blog/` with routes, migrations, and a typed config struct.

7. **Register the module**

   Add your module to `cmd/app/main.go`:

   ```go
   k.Register(blogmod.New())
   ```

   The kernel handles dependency resolution, migration execution, and route mounting automatically.

</Steps>

## What Just Happened?

When you ran `make dev`, the kernel:

1. Loaded config from `config/config.toml` with environment variable overrides
2. Created infrastructure adapters (database, cache, mail, storage, router)
3. Registered all modules and resolved dependencies via topological sort
4. Ran pending migrations for each module in dependency order
5. Initialized each module, wiring event subscriptions and services
6. Mounted HTTP routes and started the server
