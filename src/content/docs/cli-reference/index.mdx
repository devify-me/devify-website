The Devify CLI provides 19 built-in commands organized into 6 categories, plus dynamic module commands.

## Project Lifecycle

### devify init \<name\>

Scaffold a new Devify project with full interactive setup.

```bash
devify init my-app
```

Walks you through a series of prompts to configure your project:

| Step | Prompt | Options |
|------|--------|---------|
| 1 | Database engine | PostgreSQL (default), MySQL/MariaDB, SQLite, MongoDB, None |
| 2 | Cache engine | Memory (default), Redis, BadgerDB |
| 3 | Mail provider | SMTP (default), Resend, SendGrid, Postmark, SES, None |
| 4 | Storage backend | Local (default), S3, MinIO, SFTP, WebDAV |
| 5 | Renderer | Templ (default), Go templates |
| 6 | Modules | Multi-select checklist — auth pre-selected |
| 7 | OAuth providers | Google, GitHub, Discord (if auth selected) |
| 8 | PWA capabilities | Offline, install prompt, camera, GPS, etc. (if PWA selected) |
| 9 | Frontend | None, HTMX + Templ, React, Vue, Svelte, SolidJS |
| 10 | Example module | Yes / No |

**Module dependency resolution:** When you select a module, its dependencies are automatically included. For example, selecting billing auto-adds auth and tenant.

**What gets generated:**
- `cmd/app/main.go` — wired with all selected modules and adapters
- `config/config.toml` — configured for all selected adapters
- `compose.yml` — containers for database, cache, Mailpit, MinIO as needed
- `Makefile` — build, dev, test, services targets (auto-detects Podman/Docker)
- `go.mod` and `go.sum`
- `.gitignore`
- Frontend files (if HTMX or SPA selected)

### devify build

Build the application binary.

```bash
devify build
```

Runs `templ generate` (if Templ templates exist) followed by `go build ./cmd/app/`. Produces a single binary.

### devify dev

Start the development server with hot reload.

```bash
devify dev
```

Wraps Air for automatic rebuilding on file changes. Watches `.go`, `.templ`, and `.toml` files.

### devify compose

Generate `compose.yml` from `config/config.toml`.

```bash
devify compose
```

Reads the current configuration and generates a `compose.yml` with containers for the selected database engine, cache adapter (if Redis), Mailpit (in development mode), and MinIO (if storage is MinIO). Also includes any custom services declared via `[services.*]` config sections. Regenerate after changing adapters in config.

### devify run

Run the compiled binary in production mode.

```bash
devify run
```

## Module Management

### devify module:create \<name\>

Scaffold a new module.

```bash
devify module:create billing
devify module:create billing --routes --migrations --events --config
```

Interactive mode asks which capabilities to include. Flag mode skips prompts.

### devify module:list

List all registered modules with their dependencies and capabilities.

```bash
devify module:list
```

### devify module:remove \<name\>

Remove a module. Validates that no other modules depend on it before removal.

```bash
devify module:remove billing
```

## Database & Migrations

### devify migrate

Run all pending migrations in dependency order.

```bash
devify migrate
```

### devify migrate:create \<module\> \<name\>

Create a new migration file for a module.

```bash
devify migrate:create billing create_invoices
```

### devify migrate:rollback \<module\>

Rollback the last migration batch for a module.

```bash
devify migrate:rollback billing
```

### devify migrate:status

Show migration status for all modules.

```bash
devify migrate:status
```

## Code Generation

### devify gen:handler \<module\> \<name\>

Generate a handler stub.

```bash
devify gen:handler billing create-invoice
```

### devify gen:service \<module\> \<name\>

Generate a service interface and implementation.

```bash
devify gen:service billing payment
```

### devify gen:repository \<module\> \<name\>

Generate a repository interface and implementation.

```bash
devify gen:repository billing invoice
```

### devify gen:event \<module\> \<name\>

Generate an event type and subscriber.

```bash
devify gen:event billing payment-received
```

## Configuration

### devify config:validate

Validate all configuration files and module config structs.

```bash
devify config:validate
```

### devify config:dump

Dump the resolved configuration (TOML + env overrides).

```bash
devify config:dump
```

## Module Commands

Modules can register their own CLI commands via the `HasCommands` interface:

```bash
devify <module>:<command> [args]
```

Examples:

```bash
devify auth:create-admin admin@example.com password123
devify auth:list-users
devify rbac:assign-role admin@example.com admin
devify rbac:list-roles
devify tenant:list
devify tenant:create "My Company" owner@example.com
```

## Utility

### devify version

Print the CLI version.

```bash
devify version
# devify v0.1.0 — modular-monolithic Go framework CLI
```
